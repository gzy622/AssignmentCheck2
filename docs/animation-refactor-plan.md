# 过渡动画重构计划

## 目标

- 统一非打分界面过渡动画在各平台上的表现，不再为 Android 单独走一套全屏模态动画。
- 解决 Android 端打开全屏模态框时的明显卡顿，优先保证流畅和稳定，再兼顾美观。
- 保留当前“显示动画”开关语义，关闭时仍然能完全禁用动画。
- 不修改数据结构、存储格式和打分界面 `ScorePad` 的交互与动画。

## 范围

本次重构只覆盖非打分界面的过渡动画，包括：

- 顶部菜单下拉层 `dropdown`
- 统一模态框 `Modal`
- 全屏模态框 `dialog.full`
- 普通页面模态框 `dialog.page`
- 底部面板 `BottomSheet`
- 提示条 `Toast`

明确不在本轮改动内的内容：

- `ScorePad` 相关动画和交互
- 学生卡片按压、保存闪烁等局部反馈动画
- 业务流程和数据逻辑

## 现状问题

### 1. 平台表现不一致

- 当前全屏模态框在非 Android 平台使用“背景淡入 + 卡片上浮 + 导航/内容分层错峰进入”。
- Android 平台会强制切到 `lite-transitions`，变成纯淡入，视觉语言和其他平台不一致。

### 2. 动画成本偏高

- 全屏模态框存在多层并行动画：遮罩、卡片、导航、正文块分别执行。
- 正文区还带有分块错峰延迟，大列表或复杂内容打开时，首帧更容易抖动。
- Android 上虽然降级为纯淡入，但代码分支更多，后续维护和调优成本更高。

### 3. 视觉节奏偏散

- 不同组件各自使用不同的时长、位移和缓动。
- 菜单、模态框、底部面板、Toast 之间缺少统一的“运动语言”。

## 新动画设计方向

### 总体原则

- 用统一的运动参数组织全站过渡：时长、位移、透明度和缓动集中定义。
- 只使用性能稳定的属性：`opacity`、`transform`。
- 尽量避免缩放和多层级错峰，减少安卓中低端设备上的合成压力。
- 动画短一些、轻一些、收得更快，观感以“清楚出现、快速稳定”为主。

### 建议的统一语言

- 菜单、普通模态、全屏模态、Toast：
  - 统一采用“轻微纵向位移 + 淡入/淡出”。
  - 位移控制在 `8px` 到 `16px` 之间，不再使用明显的缩放。
- 底部面板：
  - 保留从底部进入的方向感，但缩短位移路径和总时长。
  - 遮罩淡入和面板上滑节奏与模态框保持同一套缓动体系。
- 全屏模态内部：
  - 取消导航和内容块逐层错峰动画。
  - 整个页面内容作为一个表面统一进入，避免复杂页面打开时“动画在跑，内容还在渲染”的撕裂感。

### 目标观感

- 打开：快速、轻、稳定，页面像一层薄面轻轻落位。
- 关闭：更短、更干脆，不拖尾。
- Android 与非 Android 视觉保持一致，只允许性能参数一致，不保留平台特化动画分支。

## 实施方案

### 第一阶段：建立统一动画令牌

在 `styles.css` 中集中定义新的运动变量，例如：

- 基础时长：`--motion-fast`、`--motion-base`、`--motion-slow`
- 进入/退出缓动：`--motion-ease-enter`、`--motion-ease-exit`
- 轻位移距离：`--motion-shift-sm`、`--motion-shift-md`
- 遮罩透明度与层级节奏变量

要求：

- 模态框、菜单、Toast、BottomSheet 统一引用，不再各自硬编码一组数值。
- 新变量命名保持和现有 CSS 变量风格一致。

### 第二阶段：重构 `Modal` 进入和退出动画

目标：

- 移除 Android 专用 `lite-transitions` 分支。
- 普通模态和全屏模态共用同一动画体系，仅保留布局层面的差异。

具体策略：

- `dialog` 遮罩继续独立淡入淡出，但时长缩短。
- `.modal-card` 改为统一的 `translateY + opacity` 动画，不再使用 `scale`。
- `dialog.full` 内部的 `.st-nav` 和 `.st-scroll-area > *` 不再逐块错峰。
- 全屏内容改为单一容器进场，必要时只保留一层极轻的补充位移。
- 保留 `is-preopen -> is-open -> is-closing` 这一状态机，不改现有打开关闭调用方式。

### 第三阶段：统一底部面板与其他过渡语言

目标：

- `BottomSheet` 保留底部抽屉属性，但运动参数与全站统一。

具体策略：

- 缩短面板进入和退出时长。
- 保留 `translateY` 驱动，不引入缩放和额外子元素动画。
- 遮罩淡入参数改为与模态框同一体系。
- 拖拽态仍然直接取消 transition，保证手势跟手。

### 第四阶段：统一轻量组件动画

覆盖组件：

- `dropdown`
- `toast`

具体策略：

- `dropdown` 从当前单独 `fadeIn` 调整为统一的 `fade + small translateY`。
- `toast` 保持现有轻提示方向，但收敛到统一时长和缓动。
- 只保留最必要的视觉反馈，不叠加多余效果。

### 第五阶段：清理平台分支与无效复杂度

需要处理的点：

- 删除 `Device.useLiteFullscreenTransitions()` 的动画特化职责。
- 删除 `body.lite-transitions` 相关 CSS 和 `modal.js` 中对应分支。
- 重新检查 `pointer guard` 和关闭等待时间，确保它们与新时长一致。

说明：

- 如果 `Device` 仍有其他用途，保留对象本身，只删除动画分支相关逻辑。
- Android Firefox 的输入聚焦稳定逻辑不属于动画分支，不在本轮随意移除。

## 性能优化要求

这次重构不只是改视觉，还要同时压低卡顿风险：

- 只动画 `transform` 和 `opacity`。
- 减少并行动画层数，特别是全屏模态的内部层。
- 避免大量子元素错峰进入。
- 保持打开时已有的延迟渲染策略，不把内容渲染压力重新堆到动画首帧。
- 关闭时长与 JS 清理计时统一，避免视觉已结束但 DOM 仍挂住，或者视觉未结束就被提前清理。

## 文件落点

预计涉及的主要文件：

- `styles.css`
- `modal.js`
- `bottom-sheet.js`
- `core.js`
- `app.js`
- `index.html`

说明：

- `index.html` 只用于同步版本号文本。
- `app.js` 主要确认“显示动画”开关仍与新体系兼容。
- `core.js` 只处理 Android 动画特化逻辑的清理。

## 验收标准

- Android、iOS、桌面浏览器打开同类页面时，动画语言一致。
- 全屏模态框打开时不再出现明显的内容分层卡顿感。
- 动画整体更短、更稳，关闭动作干净。
- “显示动画”关闭后，相关组件都能直接显示和关闭。
- 不影响 `ScorePad` 行为，不影响业务流程。

## 实施顺序建议

1. 先抽离统一动画变量和关键帧。
2. 再重构 `Modal`，优先解决全屏模态卡顿和平台分支问题。
3. 接着同步 `BottomSheet`、`dropdown`、`toast`。
4. 最后清理旧 CSS、旧分支和计时常量，并补测试。

## 风险与注意事项

- 如果只改 CSS，不同步调整 JS 中的关闭等待时间，容易出现提早卸载或残留点击穿透问题。
- 如果仍保留全屏模态内部多层错峰，即使统一平台表现，也难以真正解决 Android 卡顿。
- 如果把过渡压得太短，会显得突兀；如果压得不够，又会继续放大掉帧感，需要实际对比调节。

## 结论

本次重构建议采用“统一、轻量、去平台分支”的方案：  
以一套统一的 `opacity + translateY` 运动语言覆盖非打分界面动画，去掉 Android 专用全屏降级分支，减少层级和错峰，优先解决卡顿，再保留必要的层次感和精致度。
