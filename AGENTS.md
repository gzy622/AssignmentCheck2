# AGENTS.md

本项目智能体修改规则入口，确保修改一致、可验证、低风险。

## 项目概览

轻量化课堂作业登记工具，纯前端实现，直接打开 `index.html` 运行。

## 输出要求

- 修改仓库文件时，回复末尾附带 git commit message
- 简述分点写清主要改动
- 使用简体中文，回复正文平铺直叙、自然流动
- 不涉及修改时无需提供
- 影响版本号时同步更新 `index.html` 版本文本

## 优先阅读

修改前优先阅读：

- `index.html`：应用入口
- `boot.js`：启动入口
- `app.js`：核心状态与逻辑
- `core.js`：数据、存储与公共工具
- `actions.js`：业务操作分发
- `action-views.js`：UI 视图生成

## 关键约束

- 不引入构建工具，保持纯前端运行
- 数据存储仅使用 `localStorage`
- 不修改现有数据结构、备份文件格式和版本号机制
- 每次对项目代码进行修改时必须修改版本号
- 影响版本号时同步更新 `index.html` 版本文本

## 修改原则

- 先读相关文件，再动手修改
- 优先最小改动，复用已有函数
- 避免新增全局状态和重复逻辑
- 新功能必须补测试

## 测试与验证

仅当用户明确要求时执行并提及测试：

- `npm run test`
- `npm run test:e2e`
- 直接打开 `index.html` 运行
- 本地数据读写正常

更全面验证可运行：

```bash
npm run test:all
npm run test:coverage
```

## 沙箱与提权约定

- 执行 `npm run test:e2e` 和 git 操作时直接申请提权
- 申请提权后优先重跑失败命令
- 明确区分沙箱和提权环境的测试结果
- 用户要求提交时完成实际提交

## 测试重点

- 优先覆盖 `core.js` 和状态管理逻辑
- 涉及交互流程时补充 E2E 测试

## 完成标准

- 功能可运行
- 不破坏现有功能
- 用户要求测试时全部通过
