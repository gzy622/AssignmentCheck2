# AGENTS.md


适用于进入 `AssignmentCheck2` 仓库的任何 AI 智能体。


## 项目定位


- 这是一个课堂作业登记工具
- 核心目标是快速登记完成状态、分数、统计结果
- 无后端、无构建流程、无包管理器
- 数据保存在浏览器 localStorage


## 代码结构


- 核心文件是 index.html
- 这是单文件静态应用：HTML、CSS、JavaScript 都在同一个文件内
- README.md 用于补充功能说明


## 技术选择


- 默认使用 HTML + CSS + Vanilla JavaScript
- 默认使用 PowerShell 作为命令行工具
- 搜索代码优先使用 rg
- 不要默认引入 React、Vue、TypeScript、Node 构建链、后端或数据库


除非用户明确要求重构，否则应直接在 [index.html](/c:/codexapp/AssignmentCheck2/index.html) 上做增量修改。


## 代码修改规则


- 先理解现有实现，再修改
- 优先小改，不要无故重写整页
- 保持现有 UI、交互和数据结构稳定
- 新功能优先复用现有状态与弹窗逻辑
- 涉及数据结构时，必须同时考虑导入导出兼容性
- 涉及移动端输入或弹窗时，必须注意现有视口兼容逻辑


## 回复风格限制


回复用户时，不准使用以下风格：


- 执行日志化叙述
- 第一人称代理语气
- 工具调用与自然语言交织
- 强任务导向与阶段确认
- 高密度技术语汇


回复应满足：


- 直接说结果
- 语言简短、自然、克制
- 只保留必要技术细节
- 不写“先做什么，再做什么，最后做什么”这类流程播报
- 不写“我已经处理”“我接下来会”这类代理口吻


## 修改后汇报规则


每次完成代码或文件修改后，都要附上一条标准 Git message 风格的单行总结，用于向用户汇报本次变更。


这条 Git message 必须单独放在代码块内展示。


这条 Git message 默认使用中文书写。


允许的前缀示例：


- feat:
- fix:
- docs:
- refactor:
- style:
- test:
- chore:


这只是回复格式要求，不等同于必须执行 `git commit`。
