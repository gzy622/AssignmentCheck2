### **Project: Student Tracker Pro (HTML/JS/LocalStorage)**

#### **1. Data Structure (Core)**

- **Storage Key**: `tracker_db`
- **Config**: `const ROSTER = ["101 Name", "102 Name", ...];` (Fixed Array)
- **Schema (JSON Array of Assignments)**:

JSONdownload

```
[
  {
    "id": 1715600123,  // Timestamp ID
    "name": "Unit 1 Quiz",
    "records": {
      "101 Name": { "done": true, "score": 95 },
      "102 Name": { "done": false, "score": null }
    }
  }
]

```

#### **2. UI Layout**

- **Header**:

  - `<select>` dropdown to switch between assignments.
  - `[+]` Button to add new assignment (prompt for name).
  - `[Stats]` Button to open Statistics Modal.
- **Main View (Grid System)**:

  - Render `ROSTER` as flex/grid cards.
  - **Card State**:

    - Default: Grey/White.
    - Done (`done: true`): Green.
    - Has Score (`score !== null`): Show Score badge.

#### **3. Interactions**

- **Add Assignment**: Create new object in array -> Switch to it.
- **Student Card**:

  - **Click**: Toggle `done` status (True/False). Auto-save.
  - **Long Press (500ms)**: Open `prompt` to input Score.

    - If Score entered: Set `done: true` automatically.
- **Delete Assignment**: Small option next to dropdown to delete current view.

#### **4. Statistics Logic (Modal)**

- **Input**: Checkbox list of all existing assignments (default: all checked).
- **Global Output**:

  - `Total Submission Rate` (Total Done / Total Expected Records).
  - `Average Score` (Sum of existing scores / Count of scores).
- **Individual Output (Table)**:

  - Columns: Student Name, Personal Submission Rate (%).
  - **Sorting Controls**:

    1. By ID (Default).
    2. Rate Ascending.
    3. Rate Descending.

#### **5. Tech Constraints**

- Single HTML file (HTML + CSS + JS).
- Use `localStorage` for persistence.
- No external frameworks (Vanilla JS only).
- Use `mousedown`/`touchstart` and `mouseup`/`touchend` with `setTimeout` to detect Long Press.
